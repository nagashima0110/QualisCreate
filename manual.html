第1章　はじめに

1.1　QualisCreate とは

QualisCreate は、テストエンジニアがテスト設計を効率的かつ網羅的に行えるよう支援する ローカルデスクトップツールです。仕様書を読んで機能を分解し、テスト観点を整理し、 適切なテスト技法を適用してテストケースを生成、最終的に Excel ファイルとして出力するまでの 一連の作業を一つのツールで完結できます。

📌 このツールでできること
  • 仕様書から読み取った機能・要素をツリー構造で管理
  • 30種のテスト観点チェックリストによる観点の抜け漏れ防止
  • 11種のテスト技法によるテストケースの自動生成
  • 生成したテストケースの Excel（.xlsx）出力

1.2　動作環境

項目要件
OSWindows 10 / Windows 11
ランタイムNode.js v18 以上（Electron に同梱）
ディスク空き容量200 MB 以上
インターネット接続不要（完全ローカル動作）

1.3　起動方法

ツールフォルダ内でターミナル（PowerShell など）を開き、以下のコマンドを実行します。

cd D:\Works\02.業務\claude\QualisCreate
npx electron .
💡 ヒント npm start コマンドでも同様に起動できます。起動時に前回の作業状態が自動的に復元されます。

1.4　画面構成

画面は大きく ヘッダー と メインエリア で構成されます。

📐 ヘッダー
  • タブナビゲーション：①機能分解 → ②テスト観点 → ③テスト技法 → ④出力 の4ステップ
  • 💾 保存ボタン：現在の作業状態を JSON ファイルに保存
  • 📂 読込ボタン：保存済み JSON ファイルを読み込む

第2章　基本操作フロー

QualisCreate は以下の4ステップで使用します。各ステップは画面上部のタブで切り替えます。

① 機能分解
機能・要素を入力
›
② テスト観点
30観点をチェック
›
③ テスト技法
技法でケース生成
›
④ 出力
Excel で書き出し
  1. 機能分解（①）：仕様書を読み込み、機能と各機能に属する要素を入力します。 ここで登録した機能・要素がすべての後続ステップで対象として使われます。
  2. テスト観点チェック（②）：機能や要素ごとに 30 種のテスト観点から 該当するものにチェックを入れます。チェックした観点に基づいて推奨テスト技法が提示されます。
  3. テスト技法（③）：推奨された（または任意の）技法を選択し、 各技法の入力フォームにパラメーターを入力してテストケースを生成・追加します。
  4. 出力（④）：生成したテストケースを確認し、Excel ファイルとして出力します。
💡 ステップをスキップする場合 タブは自由に切り替え可能です。②テスト観点をスキップして直接③テスト技法から始めることもできますが、 推奨技法の表示や観点の紐づけが行われないため、①→②→③→④の順に作業することを推奨します。

第3章　① 機能分解

仕様書を読んで、テスト対象のシステムを 「機能」 と各機能に属する 「要素」 に分解して入力します。

📝 機能と要素の関係

例：機能「ユーザー登録」→ 要素「ユーザー名入力欄」「メールアドレス入力欄」「パスワード入力欄」

機能は大分類、要素は機能内の個々の入力・操作・画面項目などに相当します。

3.1　機能を追加する

  1. 左ペイン上部の 「＋ 機能追加」 ボタンをクリックします。
  2. 入力フォームが展開されます。機能名（必須）と説明（任意）を入力します。
  3. 「追加」 ボタンをクリックすると機能一覧に追加されます。

3.2　要素を追加する

  1. 左ペインの機能一覧から、要素を追加したい 機能カードをクリック して選択します。
  2. 右ペインに「要素一覧」が表示され、上部の 「＋ 要素追加」 ボタンが有効になります。
  3. ボタンをクリックし、要素名と説明を入力して 「追加」 をクリックします。
⚠ 注意 機能が選択されていない状態では「＋ 要素追加」ボタンは無効（グレーアウト）になります。 必ず先に機能を選択してから要素を追加してください。

3.3　編集・削除

各機能カード・要素カードの右端にある 「✏ 編集」 ボタンでモーダルが開き、 名前と説明を変更できます。「🗑 削除」 ボタンで削除します。

⚠ 機能削除時の注意 機能を削除すると、その機能に属する全要素と、その機能・要素に対して選択したテスト観点、 生成したテストケースもすべて削除されます。

第4章　② テスト観点チェック

30 種のテスト観点から、対象の機能・要素に対してテスト時に意識すべき観点を選択します。 選択した観点に基づいて、推奨するテスト技法が画面下部に表示されます。

4.1　テスト観点の選択手順

  1. 画面上部の 「対象機能」 ドロップダウンで機能を選択します。 （機能が１つしかない場合は自動選択されます）
  2. 必要に応じて 「対象要素」 ドロップダウンで特定の要素に絞り込みます。 「全体（機能単位）」のままにすると機能全体に対する観点として記録されます。
  3. 30 種の観点カードが表示されます。該当する観点の チェックボックス にチェックを入れます。 チェックを入れた観点は背景色が変わり視覚的に区別されます。
  4. 画面下部の 「推奨テスト技法：」 バーにチェックした観点に対応する技法が表示されます。
📌 ボタン操作
  • 全選択：全30観点にチェックを入れます。
  • チェッククリア：全チェックを解除します。
💡 機能単位と要素単位の使い分け 機能全体に共通する観点（例：負荷の観点、セキュリティの観点）は機能単位で設定し、 個別の入力項目に関する観点（同値・境界値など）は要素単位で設定するのが効果的です。

第5章　③ テスト技法

テスト技法を選択し、入力フォームにパラメーターを設定してテストケースを生成します。 ②テスト観点で選択した観点に基づき、推奨技法がハイライト表示されます。

5.1　技法の選択と画面構成

技法タブは左右2ペインで構成されます。

📐 左ペイン（技法選択サイドバー）
  • 対象機能 / 対象要素：テストケースに紐づける機能・要素を選択
  • 推奨技法：②テスト観点で選択した観点に対応する技法がチップ表示
  • 技法一覧：11種の技法ボタン。推奨技法には「推奨」バッジが付きます
📐 右ペイン（技法フォーム）
  • 意識すべきテスト観点バー：②で選択した観点が一覧表示。この技法に関連する観点はオレンジ色でハイライト
  • 入力フォーム：各技法に対応したパラメーター入力欄
  • テストケース生成ボタン：入力内容からテストケースを生成
  • テストケース追加ボタン：生成結果を確認後、テストケース一覧に追加

5.2　11のテスト技法

⚖ 同値分割
処理が同じとみなせるグループ（同値クラス）から代表値を選んでテストします。有効同値クラスと無効同値クラスに分けて設計します。
関連観点：同値の観点、動作しない観点
📏 境界値分析
同値クラスの境界上の値とその前後をテストします。バグが発生しやすい境界値付近を重点的にカバーします。
関連観点：境界値の観点、時間の観点、資源の観点
📋 デシジョンテーブル
複数の条件の組み合わせとアクション（期待動作）の関係を表で整理してテストします。条件が多い複雑なロジックに適しています。
関連観点：論理の観点、優先の観点、矛盾設定の観点
🔀 All-Pairs法
すべての因子間の2因子ペア組み合わせをカバーする最小のテストセットを自動生成します。組み合わせ爆発を防ぎながら網羅性を確保します。
関連観点：組み合わせの観点
🔢 直交法
直交表（L4〜L18）を用いて因子の水準の組み合わせを効率的に選出します。全水準を均等に試験でき、品質の偏りを防ぎます。
関連観点：組み合わせの観点
🔄 状態遷移テスト
状態・イベント・遷移先の関係を整理し、各遷移パスをテストします。画面遷移やモード切り替えのある機能に適しています。
関連観点：状態の観点、順序の観点、移行動作の観点
🗂 原因結果グラフ
原因（入力条件）と結果（出力）の論理的な関係を整理してテストケースを導出します。複数入力の組み合わせが出力を決定する場合に有効です。
関連観点：論理の観点
📖 シナリオテスト
ユーザの利用シーンを想定したシナリオ（手順のストーリー）を作成してテストします。ユーザ視点での品質確認に適しています。
関連観点：ユーザの観点、タイミング・負荷・性能の観点など
🔍 エラー推測
経験と知識から発生しやすいエラーを推測してテストケースを設計します。過去の不具合事例や「こうすれば壊れそう」という直感を活かします。
関連観点：例外・異常入力・矛盾設定・切断再接続の観点など
⚠ 異常値・特異値分析
入力フィールドの型・制約から異常値（型違い・範囲外・空など）と特異値（最大値・最小値・NULL など）のテストケースを自動生成します。
関連観点：異常入力・例外・セキュリティ・意地悪の観点
🌐 CFD法（制御フロー）
処理の制御フロー（条件分岐・ループ）に沿ったテストケースを設計します。各パスが少なくとも一度実行されることを目指します。
関連観点：論理の観点

5.3　各技法の入力方法

同値分割

  1. 「＋ クラス追加」ボタンで同値クラスを追加します。
  2. 各クラスの種別（有効／無効）、説明、代表値を入力します。
  3. 「テストケース生成」をクリックして確認後、「テストケース追加」で一覧に登録します。

境界値分析

  1. 「＋ 境界追加」ボタンで境界を追加します。
  2. 各境界の説明と境界値を入力します。ツールが境界値とその前後の値を自動生成します。
  3. 「テストケース生成」→「テストケース追加」で登録します。

デシジョンテーブル

  1. 「条件」を追加し、各条件の名前と True/False の説明を入力します。
  2. 「アクション（期待結果）」を追加します。
  3. 「テストケース生成」をクリックすると条件の組み合わせ表が生成されます。
  4. 各ルール列でどのアクションが適用されるかをチェックボックスで選択します。
  5. 「テストケース追加」で一覧に登録します。

All-Pairs法

  1. 「＋ 因子追加」ボタンで因子を追加します（例：OS、ブラウザ、言語設定）。
  2. 各因子の水準をカンマ区切りで入力します（例：Windows,Mac,Linux）。
  3. 「テストケース生成」をクリックするとペアワイズカバレッジを満たす最小テストセットが生成されます。

直交法

  1. 因子と各水準をカンマ区切りで入力します。
  2. 因子数・水準数に最適な直交表（L4〜L18）が自動選択されます。
  3. 「テストケース生成」→「テストケース追加」で登録します。

状態遷移テスト

  1. 「＋ 状態追加」で状態を追加します（例：ログアウト中、ログイン中）。
  2. 「＋ 遷移追加」で遷移を追加します。元状態・イベント・遷移先状態を指定します。
  3. 「テストケース生成」をクリックすると各遷移がテストケースとして生成されます。

原因結果グラフ

  1. 「＋ 原因追加」で入力条件を追加します。
  2. 「＋ 結果追加」で出力・アクションを追加します。
  3. 原因と結果の関係をチェックボックスで指定します。
  4. 「テストケース生成」→「テストケース追加」で登録します。

シナリオテスト

  1. 「＋ シナリオ追加」でシナリオを追加します。
  2. 各シナリオの名前、前提条件、手順（改行区切り）、期待結果を入力します。
  3. 「テストケース追加」で一覧に登録します。

エラー推測

  1. 「＋ エラー追加」でエラーケースを追加します。
  2. 推測するエラーの内容・操作・期待する動作（エラーメッセージ表示など）を入力します。
  3. 「テストケース追加」で一覧に登録します。

異常値・特異値分析

  1. 「＋ フィールド追加」で入力フィールドを追加します。
  2. フィールド名、型（文字列／数値／日付など）、制約（最大長・範囲など）を入力します。
  3. 「テストケース生成」をクリックすると型・制約に基づいた異常値・特異値ケースが自動生成されます。

CFD法（制御フロー）

  1. 「＋ ノード追加」で処理ノード（開始・処理・条件分岐・終了）を追加します。
  2. ノード間の接続（フロー）を定義します。
  3. 「テストケース生成」をクリックすると制御フローパスに沿ったケースが生成されます。

「機能フィルター」と「技法フィルター」のドロップダウンで表示するテストケースを絞り込めます。 絞り込み中の件数は右隣に表示されます。

6.2　Excel 出力

  1. 「📥 Excelで出力」ボタンをクリックします。
  2. ファイル保存ダイアログが開きます。保存先とファイル名を指定してください。
  3. Excel ファイル（.xlsx）が生成されます。機能ごとにシートが分かれます。
📌 Excel ファイルの構成
  • 機能ごとに別シートが作成されます
  • 各シートには No.・機能・要素・テスト観点・技法・条件・データ・期待結果の列があります
  • ヘッダー行は青背景・白文字で自動フォーマットされます
⚠ 注意 「🗑 全テストケース削除」ボタンをクリックするとすべてのテストケースが削除されます。 この操作は取り消せません。削除前に必ず Excel 出力しておくことを推奨します。

第7章　データの保存と読込

QualisCreate はアプリ終了時に作業状態を自動保存します。 次回起動時に自動的に前回の状態が復元されます。 また、手動での保存・読込にも対応しています。

7.1　自動保存

機能・要素の追加・編集・削除、テスト観点のチェック変更、テストケースの追加・削除などの 操作を行うたびに自動的に状態が保存されます。保存先はアプリのユーザーデータフォルダです。

7.2　手動保存・読込

ヘッダー右側の 💾 ボタンで任意のファイルパスに JSON 形式で保存できます。 📂 ボタンで保存した JSON ファイルを読み込むことができます。

💡 活用シーン
  • プロジェクト単位でデータを JSON ファイルとして管理する
  • チームメンバーに作業状態を共有する（JSON ファイルを共有→読込）
  • 定期的にバックアップを取る

第8章　よくある質問

Q. 推奨技法が表示されない

A. ②テスト観点タブで機能をドロップダウンから選択してからチェックを入れてください。 機能を選択しないとチェック内容が保存されません。機能が1つしかない場合は自動選択されます。

Q. テスト観点の「意識すべき観点バー」が空欄になる

A. ②テスト観点タブで機能・観点を選択した後、③テスト技法タブに移動すると観点が同期されます。 技法タブの「対象機能」ドロップダウンで機能を選択すると観点バーが更新されます。

Q. テストケースに機能・要素名が表示されない

A. ③テスト技法タブの左ペインで「対象機能」と「対象要素」を選択してからテストケースを生成してください。 対象を選択せずに生成・追加したケースは機能・要素が未設定になります。

Q. デシジョンテーブルで期待結果を選択できない

A. 「テストケース生成」ボタンをクリックすると条件の組み合わせ表が表示されます。 表の各列（ルール）に対して、下段のアクション行のチェックボックスで期待結果を選択してください。 その後「テストケース追加」をクリックします。

Q. Excel 出力時にエラーが発生する

A. テストケースが1件もない状態では出力できません。また、保存先フォルダに書き込み権限があるか確認してください。

Q. アプリの状態をリセットしたい

A. ④出力タブの「🗑 全テストケース削除」で全テストケースを削除できます。 機能・要素のリセットは①機能分解タブで各機能を手動で削除してください。

Q. 直交法でどの直交表が選ばれるか

A. 入力した因子数・水準数に基づいて最適な直交表が自動選択されます。 因子数や水準数が直交表の上限を超える場合はより大きい直交表が選ばれます。 全水準が2水準の場合は L4→L8→L12→L16 の順で最小テスト数のものが選択されます。
